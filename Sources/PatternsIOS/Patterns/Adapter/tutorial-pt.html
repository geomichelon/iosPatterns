Intencao

Converter a interface de uma classe para outra esperada pelo cliente.

Problema

O analytics legado tem uma API diferente da desejada pelo app.

Solucao

Crie um adapter que traduza chamadas da interface nova para a API legado.

Estrutura

  • `Analytics` define a interface alvo.
  • `LegacyAnalytics` e a classe incompatível.
  • `LegacyAnalyticsAdapter` faz a ponte entre as duas.

Aplicabilidade

  • Integrar APIs legadas ou de terceiros.
  • Manter clientes estaveis enquanto troca dependencias.

Cobertura de SOLID

  • DIP: clientes dependem da abstracao alvo (Analytics), nao da classe legada.
  • ISP: a interface alvo é focada e evita o uso de uma API legada inchada.

Pros e Contras

Pros

  • Isola dependencias legadas.
  • Permite migracao gradual para novas APIs.

Contras

  • Adiciona uma camada extra de indirecao.
  • Adapters podem crescer se as APIs divergirem muito.
