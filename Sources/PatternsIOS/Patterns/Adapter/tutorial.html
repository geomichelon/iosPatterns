Intent

Convert the interface of a class into another interface clients expect.

Problem

Legacy analytics has a different API shape than the one the app wants to use.

Solution

Create an adapter that translates calls from the new interface to the legacy API.

Structure

  • `Analytics` defines the target interface.
  • `LegacyAnalytics` is the incompatible class.
  • `LegacyAnalyticsAdapter` bridges the two.

Applicability

  • Integrating third‑party or legacy APIs.
  • Keeping clients stable while replacing dependencies.

SOLID alignment

  • DIP: clients depend on the target abstraction (Analytics), not the legacy class.
  • ISP: the target interface is focused and keeps clients from using a bloated legacy API.

Pros & Cons

Pros

  • Isolates legacy dependencies.
  • Allows gradual migration to new APIs.

Cons

  • Adds an extra layer of indirection.
  • Adapters can become large if APIs diverge a lot.
