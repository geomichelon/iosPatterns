Intent

Attach additional responsibilities to an object dynamically without changing its class.

Problem

Combining features via inheritance can explode into too many subclasses.

Solution

Wrap a base component with decorator objects that add behavior incrementally.

Structure

  • `Beverage` defines the component interface.
  • Decorators wrap another `Beverage` and extend it.
  • Multiple decorators can be composed in any order.

Applicability

  • When behavior needs to be combined at runtime.
  • Alternative to deep inheritance hierarchies.

SOLID alignment

  • OCP: behavior is extended by composition instead of changing existing classes.
  • LSP: decorators conform to the same interface as the component.

Pros & Cons

Pros

  • Composable behavior with small, reusable decorators.
  • Avoids rigid inheritance chains.

Cons

  • Harder to understand with many nested decorators.
  • Debugging can be more complex.
