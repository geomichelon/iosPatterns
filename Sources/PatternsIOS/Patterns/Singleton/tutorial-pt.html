Intencao

Garantir que uma classe tenha apenas uma instancia e fornecer um ponto global de acesso.

Problema

Varias partes do app precisam de configuracao consistente. Passar a mesma instancia por toda parte fica verboso e propenso a erros.

Solucao

Centralize a configuracao em um unico objeto e exponha uma instancia compartilhada. Impeça construcao externa para garantir unicidade.

Estrutura

  • Codigo cliente acessa `AppConfig.shared` em vez de criar novas instancias.
  • O singleton guarda estado compartilhado como base URL e flags.
  • Testes podem resetar o singleton para um estado conhecido.

Aplicabilidade

  • Configuracao global que deve ser consistente no app.
  • Caches compartilhados ou telemetria/logs.
  • Tenha cautela quando injeção de dependencia for melhor para testes.

Cobertura de SOLID

  • SRP: funciona quando o singleton tem uma responsabilidade unica e pequena (ex.: configuracao).
  • DIP: costuma ser violado porque o codigo depende de uma instancia global concreta; use com cuidado.

Pros e Contras

Pros

  • Fonte unica de verdade para estado compartilhado.
  • Acesso simples em qualquer ponto do codigo.

Contras

  • Estado global pode dificultar testes se nao for resetado.
  • Pode esconder dependencias e aumentar acoplamento.
