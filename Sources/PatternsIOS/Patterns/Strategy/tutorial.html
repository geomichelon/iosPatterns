Intent

Define a family of algorithms, encapsulate each one, and make them interchangeable.

Problem

Shipping rules vary (standard vs express) and should be swapped without changing client code.

Solution

Create a strategy interface and let the context delegate computation to a chosen strategy.

Structure

  • `ShippingStrategy` defines the algorithm contract.
  • Concrete strategies implement different pricing rules.
  • `ShippingCostCalculator` delegates to the chosen strategy.

Applicability

  • Different behaviors selected by user or environment.
  • Replacing complex conditionals with strategy types.

SOLID alignment

  • OCP: new strategies can be added without modifying the context.
  • DIP: the context depends on the ShippingStrategy abstraction.

Pros & Cons

Pros

  • Eliminates large conditional branches.
  • Open to new strategies without changing the context.

Cons

  • Increases the number of types.
  • Selecting strategies can add complexity.
